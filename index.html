CID is encoded as String using Multibase.
        
After decoding Multibase, the remaining value is bytes. 
        

      
    

    

      
      

       
Self-describing protocol for expressing raw binary
       
data in a variety of base-encoding formats.
     

      

      
        

          format: "base-encoding character - base-encoded name"
        

        
 > The base-encoding character prefix is a char that represents 
        
the encoding method based on the
        
        multibase table.
        
        
 > The base-encoded name prefix is the encoding scheme.
        

        
 > Only used in the string form of the CID.
        
 > In CIDv0, hashes are always encoded in base58btc.
        
 > base32 is used by most implementations of IPFS.
        

      
    

    

      
      

        
Self-describing codecs used to decode the data retrieved from the lookup.
        
Codecs are functions that can encode/decode into/from a data model, passed as strings.
      

      

      
        

          format: "code - name"
        

        
 > The code prefix is identifier of the codec, based on an an agreed-upon 
        
        protocol table.
        
        
 > The name prefix is the encoding method, e.g. "dag-pb" is an IPLD codec.
        

        
 > Supports many types of encoding, each with their own codec identifier.
        
 > The multicodec prefix in an IPFS CID will always be an IPLD codec.
        

        
    

    

      
      

        
Self-describing hash; the original hash is prefixed
        
with the type of cryptographic algorithm and length
        
of the hash, represented as metadata.
      

      

      
        

          format: "hash type - hash value"
        

        
 > The hash type prefix is the type of hashing algorithm used.
        
 > The hash value is the actual hash value.
        

        
 > Support multiple hashing algorithms.
        
 > The number of characters in the CID depends on the cryptographic
        
   hash of the underlying content, rather than the content size.
        
 > Most CIDs will be 256bits or 32 bytes.
